\section{ภาพรวมสถาปัตยกรรมระบบ (System Architecture)}
ระบบนี้เป็นระบบที่พัฒนาขึ้นมาเพื่อตรวจวัดสภาพอากาศ เก็บรวบรวมข้อมูลจากเซนเซอร์ต่างๆ และส่งข้อมูลไปยังเซิร์ฟเวอร์
เพื่อจัดเก็บ ประมวลผล และแสดงผลผ่านหน้าเว็บไซต์ 
\subsection{ภาพรวมของโครงสร้างระบบ}
** ใส่รูปภาพโครงสร้างระบบ **
ระบบนี้ประกอบด้วย 3 ส่วนหลักๆ ดังนี้
\begin{itemize}
    \item ส่วนอุปกรณ์ตรวจวัด (Sensor Devices): ประกอบด้วยเซนเซอร์ต่างๆ ที่ใช้ในการตรวจวัดสภาพอากาศ ได้แก่ 
    อุณหภูมิบรรยากาศ ความชื้น ความกดอากาศ ความเร็วลม ทิศทางลม ความเข้มแสง และอุณหภูมิที่แผงโซลลาเซลล์
    จากนั้นส่งข้อมูลไปยังไมโครคอนโทรลเลอร์เพื่อประมวลผลเบื้องต้นและส่งต่อไปยังเซิร์ฟเวอร์
    
    \item ส่วนความคุมและประมวลผลภาคสนาม (Edge Control and Processing): ใช้ไมโครคอนโทรลเลอร์ในการรับข้อมูลจากเซนเซอร์ 
    ตรวจสอบความถูกต้อง และส่งข้อมูลไปยังเซิร์ฟเวอร์ผ่านเครือข่ายไร้สาย
    
    \item ส่วนการสื่อสาร (Communication): ข้อมูลจะถูกส่งผ่านเครื่องข่ายไร้สาย (4G LTE) ไปยังเซิร์ฟเวอร์ โดยใช้โปรโคคอล HTTP/HTTPS
    ไปยัง API ของเซิร์ฟเวอร์เพื่อจัดเก็บข้อมูลในฐานข้อมูลและประมวลผล

    \item ส่วนประมวลผล และจัดเก็บข้อมูล (Data Processing and Storage): เซิร์ฟเวอร์จะรับข้อมูลจากอุปกรณ์ตรวจวัดผ่าน API และตรวจสอบความถูกต้อง
    แล้วจึงบันทึกลงฐานข้อมูล และทำการทำนายการผลิตไฟฟ้าของโซลาร์เซลล์ล่วงหน้า 24 ชั่วโมง โดยใช้ข้อมูลที่เก็บมา

    \item ส่วนแสดงผล (Visualization): ผู้ใช้สามารถเข้าถึงข้อมูลผ่านหน้าเว็บไซต์ที่แสดงผลข้อมูลสภาพอากาศ และการทำนายการผลิตไฟฟ้าของโซลาร์เซลล์ล่วงหน้า 24 ชั่วโมง
    ในรูปแบบกราฟ ณ เวลาต่างๆ
\end{itemize}

\subsection{ลำดับการไหลของข้อมูล (Data Flow)}
กระบวรการทำงานของระบบสามารถสรุปเป็นลำดับได้ดังนี้
\begin{enumerate}
    \item เซนเซอร์ทำการวัดค่าสภาพอากาศต่างๆ
    \item ไมโครคอนโทรลเลอร์รับข้อมูลจากเซนเซอร์
    \item ไมโครคอนโทรลเลอร์ตรวจสอบความถูกต้องของข้อมูลและส่งข้อมูลไปยังเซิร์ฟเวอร์ผ่านเครือข่ายไร้สาย
    \item เซิร์ฟเวอร์รับข้อมูลผ่าน API และตรวจสอบความถูกต้องของข้อมูล
    \item เซิร์ฟเวอร์บันทึกข้อมูลลงฐานข้อมูลและทำการทำนายการผลิตไฟฟ้าของโซลาร์เซลล์ล่วงหน้า 24 ชั่วโมง
    \item ผู้ใช้เข้าถึงข้อมูลผ่านหน้าเว็บไซต์ที่แสดงผลข้อมูลสภาพ
\end{enumerate}

\subsection{แนวคิดการออกแบบระบบ}
ระบบถูกออกแบบมาให้มีคุณสมบัติดังนี้
\begin{itemize}
    \item แยกส่วนการทำงานของระบบให้ชัดเจน
    \item รองรับการขยายสภานีตรวจวัดเพิ่มเติมในอนาคตได้ง่าย
    \item มีความทนทานและเชื่อถือได้ในการส่งข้อมูลจากภาคสนามไปยังเซิร์ฟเวอร์
\end{itemize}
