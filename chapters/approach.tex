\begin{itemize}
    \item [] \textbf{3.1.2.1 การแปลงสัญญาณจาก I2C เป็น RS485 (I2C to RS485 Conversion)} \\ 
    \hspace{\parindent} จากตารางที่ \ref{tab:sensor list} จะเห็นว่าเซนเซอร์จำนวน 2 ตัว ได้แก่ BME280 และ VEML7700 ใช้การสื่อสารแบบ I2C ซึ่งเหมาะสำหรับการสื่อสารระยะสั้นภายในบอร์ดเดียวกัน 
    อย่างไรก็ตาม I2C มีข้อจำกัดด้านระยะทางและความทนทานต่อสัญญาณรบกวน จึงไม่เหมาะสมสำหรับการใช้งานในระบบที่ต้องติดตั้งเซนเซอร์ในระยะไกล
    \item [] \textbf{3.1.2.2 การแปลงสัญญาณจาก RTD เป็น RS485 (RTD to RS485 Conversion)} \\ 
    \hspace{\parindent} จากตารางที่ \ref{tab:sensor list} จะเห็นว่าเซนเซอร์ PT100 ที่ใช้ในการวัดอุณหภูมิ ณ บริเวณแผงโซลาร์เซลล์ ซึ่งเป็นเซนเซอร์ที่ทำงานโดยอาศัยการเปลี่ยนแปลงค่าความต้านทานตามอุณหภูมิ ซึ่งไม่สามารถเชื่อมต่อเข้ากับไมโครคอนโทรลเลอร์โดยตรงได้ จำเป็นต้องมีวงจรแปลงสัญญาณเพื่อแปลงค่าความต้านทานให้อยู่ในรูปแบบสัญญาณไฟฟ้าที่สามารถประมวลผลได้ \\
    
    \hspace{\parindent} ในการออกแบบระบบนี้ ได้มีการใช้โมดูลแปลงสัญญาณ RTD เป็น RS485 ซึ่งภายในโมดูลจะทำหน้าที่แปลงค่าความต้านทานของ PT100 ให้เป็นข้อมูลดิจิทัล และส่งออกผ่านการสื่อสารแบบ RS485 โดยใช้โปรโตคอล Modbus RTU ทำให้สามารถเชื่อมต่อเข้ากับระบบหลักได้โดยตรง \\
    กระบวนการทำงานของระบบสามารถอธิบายได้ดังนี้
    
    \begin{enumerate}
        \item โมดูล RTD ทำการวัดค่าความต้านทานจากเซนเซอร์ PT100
        \item แปลงค่าความต้านทานเป็นค่าอุณหภูมิด้วยวงจรภายใน
        \item จัดเก็บข้อมูลในรูปแบบรีจิสเตอร์ของ Modbus
        \item ส่งข้อมูลผ่าน RS485 เมื่อได้รับคำสั่งจากอุปกรณ์ Master
    \end{enumerate}
    
    \hspace{\parindent} การออกแบบลักษณะนี้ช่วยลดภาระการประมวลผลของไมโครคอนโทรลเลอร์ เนื่องจากการแปลงค่าทางกายภาพเป็นข้อมูลดิจิทัลถูกจัดการภายในโมดูลแล้ว อีกทั้งยังช่วยเพิ่มความแม่นยำของข้อมูล และลดความซับซ้อนในการออกแบบวงจร
    นอกจากนี้ การใช้การสื่อสารแบบ RS485 ยังช่วยให้สามารถติดตั้งเซนเซอร์ในระยะไกลได้ และมีความทนทานต่อสัญญาณรบกวน เหมาะสำหรับการใช้งานในสภาพแวดล้อมจริง

    \item [] \textbf{3.1.2.3 การสื่อสารข้อมูลด้วย RS485 และ Modbus RTU (RS485 and Modbus RTU Communication)} \\ 
    \hspace{\parindent} ภายในระบบสถานีวัดสภาพอากาศขนาดย่อม ได้เลือกใช้การสื่อสารแบบ RS485 ร่วมกับโปรโตคอล Modbus RTU เป็นกลไกหลักในการรับส่งข้อมูลระหว่างอุปกรณ์ต่าง ๆ เนื่องจาก RS485 มีความสามารถในการสื่อสารในระยะไกล และมีความทนทานต่อสัญญาณรบกวน เหมาะสำหรับการติดตั้งในสภาพแวดล้อมภายนอกอาคาร
    
    \hspace{\parindent} ในระบบนี้ โครงสร้างการสื่อสารออกแบบมาในลักษณะ Master-Slave โดยมีชุดควบคุมหลักทำหน้าที่เป็น Master และอุปกรณ์เซนเซอร์ทำหน้าที่เป็น Slave ซึ่งแต่ละอุปกรณ์จะมีหมายเลขประจำตัว (Slave ID) ดังนี้
    \begin{table}[h]
    \centering
    \begin{tabular}{|l|l|}
    \hline
    \textbf{อุปกรณ์} & \textbf{SlaveID} \\ \hline
    Wind Speed Sensor & 1 \\ \hline
    Wind Direction Sensor & 2 \\ \hline
    PT100 & 3 \\ \hline
    Sensor Node & 4 \\ \hline
    \end{tabular}
    \end{table}

    กระบวนการสื่อสารสามารถอธิบายได้ดังนี้
    \begin{enumerate}
        \item ชุดควบคุมหลัก (Master) ส่งคำสั่งอ่านข้อมูลไปยังอุปกรณ์เซนเซอร์ (Slave) โดยระบุ Slave ID และรีจิสเตอร์ที่ต้องการอ่าน
        \item อุปกรณ์เซนเซอร์ที่ได้รับคำสั่งจะทำการประมวลผลและส่งข้อมูลกลับมายังชุดควบคุมหลักในรูปแบบของเฟรม Modbus RTU
        \item ชุดควบคุมหลักทำการตรวจสอบความถูกต้องของข้อมูลที่ได้รับ และนำข้อมูลไปประมวลผลต่อไป
    \end{enumerate}
\end{itemize}

\section{เซิฟเวอร์และเว็บไซต์สำหรับแสดงผล}
\subsection{ภาพรวมของระบบ (System Overview)}
\hspace{\parindent}ระบบเซิร์ฟเวอร์และเว็บไซต์สำหรับแสดงผลถูกพัฒนาขึ้นเพื่อทำหน้าที่เป็นศูนย์กลางในการจัดการข้อมูลจากสถานีวัดสภาพอากาศ และแสดงผลลัพธ์จากโมเดลการทำนายการผลิตไฟฟ้าจากโซลาร์เซลล์ล่วงหน้า 24 ชั่วโมง โดยระบบถูกออกแบบในลักษณะของสถาปัตยกรรมแบบ Client-Server เพื่อให้สามารถรองรับการทำงานแบบเรียลไทม์และการเข้าถึงข้อมูลผ่านเครือข่ายอินเทอร์เน็ต

ในภาพรวม ระบบสามารถแบ่งออกเป็น 4 ส่วนหลัก ได้แก่ สถานีวัดสภาพอากาศ (Weather Station), เซิร์ฟเวอร์ (Server), ฐานข้อมูล (Database) และเว็บไซต์สำหรับแสดงผล (Web Application) โดยมีการทำงานร่วมกันดังนี้

\begin{enumerate}
    \item สถานีตรวจวัดสภาพอากาศขนาดย่อม ทำการตรวจวัดข้อมูลสภาพแวดล้อม และส่งข้อมูลไปที่เซิฟเวอร์ผ่านเครือข่ายอินเทอร์เน็ต
    \item เซิร์ฟเวอร์ทำหน้าที่รับข้อมูลจากสถานีตรวจวัด จัดเก็บข้อมูลลงในฐานข้อมูล
    \item เซิร์ฟเวอร์เรียกใช้งานโมเดลการทำนายเพื่อคาดการณ์การผลิตไฟฟ้าล่วงหน้า 24 ชั่วโมง โดยใช้ข้อมูลล่าสุดร่วมกับข้อมูลในอดีต
    \item ผลลัพธ์จากการทำนายและข้อมูลสภาพอากาศจะถูกเก็บลงฐานข้อมูล และส่งไปยังเว็บไซต์เพื่อแสดงผลให้ผู้ใช้งานสามารถเข้าถึงได้
\end{enumerate}

การออกแบบระบบในลักษณะนี้ช่วยให้สามารถแยกหน้าที่การทำงานของแต่ละส่วนได้อย่างชัดเจน (Modular Design) เพิ่มความยืดหยุ่นในการพัฒนา และรองรับการขยายระบบในอนาคต เช่น การเพิ่มจำนวนสถานีตรวจวัด หรือการปรับปรุงโมเดลการทำนาย

\subsection{เทคโนโลยีและเครื่องมือที่ใช้ (Technology Stack and Tools)}
\begin{itemize}
    \item[]\textbf{3.3.2.1 ฐานข้อมูล (Database)}\\
    \hspace{\parindent} ระบบเลือกใช้ PostgreSQL เป็นฐานข้อมูลหลัก เนื่องจากเป็น Open Source และรองรับการจัดเก็บข้อมูลแบบ Time-Series ได้อย่างมีประสิทธิภาพ เหมาะสำหรับเก็บข้อมูลสภาพอากาศ, ผลลัพธ์การทำนาย และข้อมูลการผลิตไฟฟ้าจากโซลาร์เซลล์

    \item[]\textbf{3.3.2.2 หลังบ้าน (Backend)}\\
    \hspace{\parindent} สำหรับ Backend ใช้ Fast API เนื่องจากรองรับการสร้าง API ที่มีประสิทธิภาพสูง สามารถทำงานร่วมกับข้อมูลและโมเดล Machine Learning ได้ดี อีกทั้งยังช่วยให้การพัฒนาระบบทำได้รวดเร็วและมีโครงสร้างที่ชัดเจน

    \item[]\textbf{3.3.2.3 หลังบ้าน (Frontend)}\\
    \hspace{\parindent} Frontend ใช้ Vite.js ร่วมกับ React เพื่อสร้างเว็บไซต์ที่สามารถรองรับทั้ง Desktop และ Mobile View พร้อมความสามารถในการเชื่อมต่อแบบเรียลไทม์กับ Backend ผ่าน API

\end{itemize}

\subsection{การออกแบบฐานข้อมูลและการจัดการข้อมูล (Database Design and Data Management)}
\begin{figure}[H]
    \centering

    \caption{แผนภาพแสดงความสัมพันธ์ของตารางฐานข้อมูล}
    \label{fig:ER diagram}
\end{figure}

\begin{itemize}
    \item[]\textbf{3.3.3.1 ตารางข้อมูล Station}

    \hspace{\parindent} ระบบเลือกใช้ PostgreSQL เป็นฐานข้อมูลหลัก เนื่องจากเป็น Open Source และรองรับการจัดเก็บข้อมูลแบบ Time-Series ได้อย่างมีประสิทธิภาพ เหมาะสำหรับเก็บข้อมูลสภาพอากาศ, ผลลัพธ์การทำนาย และข้อมูลการผลิตไฟฟ้าจากโซลาร์เซลล์

    \item[]\textbf{3.3.3.2 ตารางข้อมูล weather\_db}

    \hspace{\parindent} ตารางนี้ใช้สำหรับเก็บข้อมูลสภาพอากาศที่วัดได้จากเซนเซอร์ในแต่ละช่วงเวลา ประกอบไปด้วยค่าอุณหภูมิอากาศ (air\_temp) ความชื้นสัมพัทธ์ (humidity) ความกดอากาศ (air\_pressure) ความเร็วลม (wind\_speed) ทิศทางลม (wind\_direction) ความเข้มของแสง (light\_lux) และอุณหภูมิแผงโซลาร์เซลล์ (solar\_temp) โดยมีคีย์หลักร่วมคือ station\_id และเวลาที่ทำการบันทึกข้อมูล (recorded) เพื่อระบุชุดข้อมูลสภาพอากาศของแต่ละสถานีตามเวลาจริง

    \item[]\textbf{3.3.3.3 ตารางข้อมูล rawdata\_pv}

    \hspace{\parindent} ตารางนี้ใช้สำหรับเก็บข้อมูลปริมาณการผลิตไฟฟ้าจริงซึ่งเป็นข้อมูลที่ขอมาจากศูนย์ SCMC มหาวิทยาลัยเชียงใหม่ ประกอบไปด้วยค่าปริมาณไฟฟ้าที่ผลิตได้ (pv\_unit) ในแต่ละช่วงเวลา (timestamp) โดยมีการเชื่อมโยงผ่าน station\_id เพื่อให้สามารถนำข้อมูลการผลิตไฟฟ้าจริงไปเปรียบเทียบกับสภาวะอากาศในช่วงเวลาเดียวกันได้
    
    \item[]\textbf{3.3.3.4 ตารางข้อมูล pv\_prediction}

    \hspace{\parindent} ตารางนี้ใช้สำหรับเก็บผลลัพธ์ที่ได้จากการพยากรณ์ปริมาณการผลิตไฟฟ้าล่วงหน้า ประกอบไปด้วยเวลาที่คาดการณ์ (forecast\_time) และปริมาณไฟฟ้าพยากรณ์ (pv\_output) โดยเชื่อมโยงกับตารางสถานีผ่าน station\_id และอ้างอิงความสัมพันธ์กับตาราง weather\_db เพื่อใช้เป็นข้อมูลนำเข้าในการสร้างแบบจำลองการพยากรณ์

    \item[]\textbf{3.3.3.5 ตารางข้อมูล authorities}
    
    \hspace{\parindent} ตารางนี้ใช้สำหรับการจัดเก็บข้อมูลผู้ดูแลระบบหรือเจ้าหน้าที่ที่มีสิทธิ์ในการเข้าถึงระบบ ประกอบด้วย เลขพนักงาน (employee\_id) ชื่อพนักงาน (employee\_name) และรหัสผ่าน (password) สำหรับการตรวจสอบสิทธิ์ (Authentication) ในการเข้าใช้งานหน้าจอการบริหารจัดการข้อมูล

\end{itemize}

\subsection{การออกแบบส่วนติดต่อผู้ใช้และเว็บไซต์ (Frontend and User Interface Design)}

\hspace{\parindent}Web Dashboard นี้ถูกออกแบบโดยคำนึงถึงการแสดงข้อมูลที่จำเป็นเพื่อให้ระบบมีประสิทธิภาพ ใช้งานง่ายและตอบโจทย์ความต้องการของผู้ใช้งานดังนี้
\begin{enumerate}
    \item แสดงข้อมูลปริมาณการพยากรณ์การผลิตไฟฟ้าจากโซลาร์เซล์โดยให้ผู้ใช้สามารถเห็นถึงช่วงเวลาที่มีการผลิตไฟฟ้าจากโซลาร์เซลล์ร่วมด้วย
    \item แสดงผลข้อมูลปริมาณการผลิตไฟฟ้าได้จริงโดยโดยให้ผู้ใช้สามารถเห็นถึงช่วงเวลาที่มีการผลิตไฟฟ้าจากโซลาร์เซลล์ร่วมด้วย
    \item  แสดงข้อมูลสภาพอากาศที่ได้มาจากสถานีตรวจวัดสภาพอากาศขนาดย่อมโดยข้อมูลตัวเลขรายชั่วโมงต้องถูกนำมาแสดงผลเพื่อให้ผู้ใช้เห็นภาพแนวโน้มของสภาพอากาศ
    \item เพื่อให้ง่ายต่อการติดตามดูแลรักษาสถานีวัดสภาพอากาศขนาดย่อมควรมีการติดตามสถานะของระบบการทำงานและสถานะเซนเซอร์ว่ายังสามารถทำงานปกติหรือไม่
\end{enumerate}
