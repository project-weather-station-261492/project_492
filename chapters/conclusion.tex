\section{\ifenglish Conclusions\else สรุปผล\fi}
\hspace{\parindent} จากการดำเนินโครงงานนี้ ผู้จัดทำได้สร้างระบบพยากรณ์ปริมาณการผลิตไฟฟ้าจากแผงโซลาร์เซลล์ที่ประกอบด้วยหลายส่วนทำงานร่วมกัน ตั้งแต่การเก็บข้อมูลจากสถานีวัดสภาพอากาศ การพัฒนาโมเดลพยากรณ์ และการสร้าง Web Application สำหรับผู้ใช้งาน ซึ่งผลลัพธ์โดยรวมแสดงให้เห็นว่าระบบสามารถทำงานได้จริงและตอบโจทย์วัตถุประสงค์ของโครงงาน ดังนี้

เริ่มต้นจากการติดตั้งสถานีวัดสภาพอากาศขนาดย่อมเพื่อเก็บข้อมูล ได้แก่ อุณหภูมิ ความชื้น ความเร็วลม ทิศทางลม ปริมาณน้ำฝน ความกดอากาศ และค่าความเข้มแสง โดยตั้งให้สถานีส่งข้อมูลทุก 1 ชั่วโมง ผ่านเครือข่ายไร้สายไปยังเซิร์ฟเวอร์ จากนั้นนำข้อมูลดังกล่าวเข้าสู่ขั้นตอน จัดเก็บในฐานข้อมูลและผ่านกระบวนการ Data Processing เช่น การจัดรูปแบบเวลา การรวมข้อมูลเข้ากับข้อมูลการผลิตไฟฟ้าย้อนหลังและการคำนวณค่ามุมตกกระทบของแสงอาทิตย์กับแผงโซลาร์เซลล์ 
เมื่อได้ข้อมูลที่พร้อมใช้งานแล้วระบบจึงนำข้อมูลเหล่านี้ไปใช้สร้างและฝึกฝนโมเดลด้วยใช้เทคนิค Recurrent Neural Network (RNN) ประเภท Long Short-Term Memory (LSTM) ซึ่งเป็นเทคนิคที่เหมาะสมสําหรับการทํางานกับข้อมูลที่มีลักษณะเป็นลําดับเวลา (Time Series Data) ผลลัพธ์การทดลองโมเดลชี้ให้เห็นว่าโมเดลมีความสามารถในการทำนายในระดับที่น่าพอใจโดยมีค่า $R^2$ = 0.8619 และ MAE = 2.427 kWh ซึ่งสะท้อนว่าโมเดลสามารถทำนายการผลิตไฟฟ้าได้ค่อนข้างใกล้เคียงกับค่าจริง 
อย่างไรก็ตาม เมื่อทดสอบในสภาพการใช้งานจริงพบความคลาดเคลื่อนเฉลี่ย 4.574 kWh เนื่องจากข้อมูลสภาพอากาศที่ใช้ฝึกโมเดลไม่ใช่ข้อมูลที่ได้จากสถานีวัดทำให้มีความต่างของข้อมูลเลยเกิดความคลาดเคลี่อน ในส่วนของระบบแสดงผลได้พัฒนา Web Application ที่ผู้ใช้งานสามารถเข้าดูกราฟข้อมูลสภาพอากาศและข้อมูลการผลิตไฟฟ้าจากโซลาร์เซลล์ย้อนหลังและผลการพยากรณ์ล่วงหน้า 24 ชั่วโมง โดยผลการประเมินความพึงพอใจเฉลี่ยอยู่ที่ 4.77 คะแนน หรือคิดเป็น 95.34\%  
สะท้อนให้เห็นว่าระบบช่วยให้ผู้ใช้งานเข้าถึงข้อมูลได้ง่าย ทั้งนี้ระบบดังกล่าวสามารถนําไปประยุกต์ใช้ในการวางแผนการบริหารจัดการพลังงานภายในอาคารได้อย่างมีประสิทธิภาพ โดยสามารถช่วยให้ผู้ใช้งานสามารถลดค่าใช้จ่ายด้านพลังงานได้ในระยะยาว

\section{\ifenglish Challenges\else ปัญหาที่พบและแนวทางการแก้ไข\fi}

ในการทำโครงงานนี้ พบว่าเกิดปัญหาหลักๆ ดังนี้

\begin{enumerate}
        \item ข้อมูลค่าการผลิตไฟฟ้าย้อนหลังจากโซลาร์เซลล์บางช่วงมีค่าขาดหายไป หรือมีความไม่ต่อเนื่องของข้อมูล ทำให้ต้องมีการทำ Data Cleaning และจัดรูปแบบข้อมูลใหม่ก่อนนำไปใช้ในการฝึกฝนโมเดล
        \item รูปแบบของข้อมูลมีความแตกต่างกันเช่น ข้อมูลสภาพอากาศและข้อมูลการผลิตไฟฟ้าจากโซลาร์เซลล์ มีรูปแบบ Timestamp ที่แตกต่างกันทำให้ต้องมีการปรับรูปแบบของข้อมูลให้มีรูปแบบ Timestamp ที่เหมือนกันทั้งข้อมูลสภาพอากาศและข้อมูลการผลิตไฟฟ้า และทำการรวมข้อมูลเพื่อให้สามารถนำมาใช้ร่วมกันในโมเดลได้
        \item เนื่องจากข้อมูลการผลิตไฟฟ้าย้อนหลังจากโซลาร์เซลล์ที่มีเป็นแบบรายวันแล้วเราต้องการที่จะทำนายปริมาณการผลิตไฟฟ้าแบบรายชั่วโมง จึงต้องมีการทำการแปลงข้อมูลโดยการใช้เทคนิคการแปลงข้อมูลจากข้อมูลรายวันมาเป็นค่าการผลิตไฟฟ้าในแต่ละชั่วโมง
        \item เนื่องจากพึ่งติดตั้งสถานีวัดสภาพอากาศขนาดย่อมทำให้มีข้อมูลสภาพอากาศย้อนหลังไม่มากนัก ทำให้ต้องมีการใช้ข้อมูลสภาพอากาศจากแหล่งอื่นเพื่อใช้ในการโมเดล
\end{enumerate}   


\section{\ifenglish%
Suggestions and further improvements
\else%
ข้อเสนอแนะและแนวทางการพัฒนาต่อ
\fi
}

ข้อเสนอแนะเพื่อพัฒนาโครงงานนี้ต่อไป มีดังนี้

\begin{enumerate}
        \item เพิ่มปริมาณข้อมูลสำหรับการฝึกสอนโมเดลให้มากขึ้นเพื่อให้โมเดลมีความแม่นยำมากขึ้นในการทำนายปริมาณการผลิตไฟฟ้า
        \item พัฒนาให้ระบบสามารถทำ Real-time Prediction ได้จริง
        \item สามารถนำโมเดลที่พัฒนาขึ้นไปประยุกต์ใช้ร่วมกับระบบบริหารจัดการพลังงานเพื่อช่วยวางแผนการใช้พลังงานไฟฟ้าให้เกิดประสิทธิภาพสูงสุด
        \item เพิ่มระบบแจ้งเตือนเมื่อมีเซ็นเซอร์ตัวใดตัวหนึ่งมีปัญหาในการทำงานเพื่อให้สามารถแก้ไขได้อย่างรวดเร็ว
\end{enumerate}
